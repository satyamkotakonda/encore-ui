Basics

  • This sub-site provides development teams at Rackspace with a set of design patterns, to help ensure a consistent and unified look-and-feel for all products under the Encore umbrella.
  • This section of the Demo site focuses mainly on markup, CSS and design patterns. If you're curious about Encore-specific angular.js directives, find the "All Components" section in the left hand navigation.
  • If you're a Rackspace team outside of Encore and need help setting up an new project from an engineering perspective, go to this Getting Started guide in the EncoreUI repository.
    • For the most part, RackSpace developer teams will be creating mark-up using the rxPage subdirective of rxApp as a container.
    • rxPage's template code contains a breadcrumb component, an attribute to pre-fill titles and subtitles and a rxNotification container so you can send page-level communication errors and validation messages in a standardized way. Check the angular.js code for controllers within Cloud Encore for implementation examples.
  • The rxApp common.less file defines many base CSS rules and classes for app use. Included in this is normalize.css. This helps create a consistent starting point for styles across all browsers.
  • Kevin Lamping, a developer on the EncoreUI team, wrote a document to address coding standards while writing CSS.

Color

  • ENCORE-SPECIFIC: Color is an integral part of communicating with users of Encore. Certain colors in Encore are reserved for certain types of actions or events.
    • 24b56a
      • This green is used for links and icons that deal explicitly with create and add actions.
      • Note that this green should not be used to communicate "active" or "success", the distinction being that these are states, not actions a user can take.
    • c8e5bf
      • This green is used for statuses that communicate success.
    • 4881c0
      • This blue is used for links that direct the user to a new page or area.
    • 86cafd
      • This blue is used for statuses that are meant to be informative, but are not associated with success, failure, warning, or error.
    • dededd
      • This gray is used to communicate a button, link, or element is disabled.
    • 898989
      • This gray is used to communicate that an icon, button, or link is associated with a cancel action.
    • db7820
      • This orange is used to mean "warning", especially in cases where the user's attention or confirmation is needed before they take an additional action.
      • This orange is also used for links and icons that deal with editing, changing, or updating something. In some cases this may cause an irreversible change to that object.
    • fc0f1d
      • This red is used to mean "error", either as part of a response or status message.
      • Red is also used for link colors and icons associated with destructive/destroy actions that a user can take. Note that irreversible changes (like rebuilding a server) are not considered destructive actions.

Typography

  • ENCORE-SPECIFIC: We use Google Fonts, in particular the Roboto font. While Google Fonts are not included in the EncoreUI package, they are included if you use the encore-ui-template repository as a base.
  • For developers not using encore-ui-template, read the README.md file for rxApp on fonts for more information.

Flexbox Grid

  • Encore UI incorporates the Layout module from the Angular Material framework to provide an assortment of attribute based layout options based on the flexbox layout model. Included are intuitive attribute based styles that ease the creation of responsive row and/or column based page layouts. Visit the layout section for info on how to use this attributes.

Heading/Title Styles

  • Headings are sized using the title class and an additional xl, lg, sm and xs classes. Not adding the additional class defaults the size to medium.
  • The size of the heading is not impacted by the heading level used (e.g. <h1>, <h2>). This is done to ensure that the heading levels are sequential.
  • ENCORE-SPECIFIC: For design consistency, we usually use the default medium size (<h2 class="title">) for most subheaders. Other headers classes can be used for context.

Descriptions / Metadata

  • The EncoreUI metadata component is a list of terms with their associated descriptions. This is used throughout Encore to show metadata, especially on an individual object such as a server or a CBS volume.
  • ENCORE-SPECIFIC: This is an ideal solution if you have anywhere from 5-10 definitions. (See a Server Details page or Image Details page under Accounts -> Cloud for an example.) Anything more - or anything displaying visual or image content - and you may want to consider a different design pattern. (See the "Non-Data based Tables" section in the Table page for more information.)

Lists

  • Lists by default do not have styles. Add bullets/decimals to lists with the list class. It is also possible to build non-semantic lists using the additional item class.
  • ENCORE-SPECIFIC: Use this in conjunction with icons, colors and helper classes such as .flush-right to build out action menus. Look at the detail pages in Cloud Servers and view-source for additional context.

Wells

  • The well class creates a well effect - a box with significant padding and light gray background.
  • Three well styles are available: one for modals, one for left-aligned summary text on a full page, and one for providing an explanation as to why data is not available to the user.
  • Modal wells: Modal wells are used in the context of modals, typically to provide a summary of data entered or decisions made by the user in a form to provide a clearer picture to the user of changes they are about to submit. Use the class modal-well for this type of well.
  • Left-aligned wells: These wells are used in conjunction with forms in much the same way modal wells are utilized, except left-aligned wells appear in a full-page context. Use the class well for this type of well.
  • Center-aligned wells: These wells are used to communicate why data is not present for the user, usually if data can only be seen with a permissions the user has not been granted. In other cases, it may be used to prompt the user to add data to an empty notes field. When utilizing this kind of well, list items may be appropriate. Use the classes well and centered for this type of well. Use the class list on a ul for correct styles and alignment.
  • ENCORE-SPECIFIC: In the context of Encore, we currently use this in the context of adding large amounts of extraneous or optional information. (Take a look at Account -> Cloud -> Database -> Create New Instance for an example of this or view source.)

Helper Classes

  • Subdued Text: The .subdued class is a basic color change to make text less prominent. The style persists if the class is nested in a link.
  • Full Width: .full-width class that marks content as full width (e.g. to give a button full width). Requires 'display' to be defined as inline-block or block.
  • Hidden content
    • .hidden class - Will add to hide an element but not remove it from the spacing of the page.
    • .visually-hidden class - Used to hide content visually but can still be read by screen readers (i.e. removed from the page spacing)
  • Clearing floats
    • .clear class - Provides clearfix functionality See http://nicolasgallagher.com/micro-clearfix-hack.
    • .clear-left, .clear-right classes - Simple classes to help with clearing previous sibling floats

Progress Bars

  • There are two states for progress bars: active and complete. Active progress bars have animated stripes, which are provided by the class .active. Full progress bars do not have this class, and are solid and static in appearance.
  • Several options for displaying text with a progress bar are available: no text, a percentage (not italicized), and detailed text (italicized). Showing a percent is preferred to showing nothing.
